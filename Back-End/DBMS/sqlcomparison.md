# SQL 절 정의 및 비교 (=, IN, OR)

**조건문의 핵심 요소**로, `=`**는 단일 값과 정확히 일치하는 행**을 찾고, `IN`**은 주어진 목록(값들의 집합) 중 하나와 일치하는 행**을 찾는 차이  
(`IN`은 여러 `OR` 조건을 효과적으로 처리하거나 성능 개선에 도움을 주지만, 값의 개수가 많을 때는 주의)

### **SQL **`=`** 절 (단일 값 비교)**

- **용도**: 특정 컬럼의 값이 지정된 단일 값과 정확히 일치하는 행을 필터링
- **예시**: `WHERE column_name = '특정값'`
- **특징**: 가장 기본적인 필터링 방법으로, 특정 하나의 값만 찾을 때 사용

### **SQL **`OR`** (논리 연산자)**

- **용도**: 여러 조건 중 **하나라도 참**이면 레코드를 반환
- **예시**: `WHERE department = 'Sales' OR department = 'Marketing'`

  (부서가 'Sales'이거나 'Marketing'인 레코드)
- **특징:**
  - WHERE 절에서 동적으로 여러 개의 조건을 추가 가능
  - OR 절에 많은 조건을 포함하면 SQL 쿼리가 길어지고 가독성 및 성능 저하

### **SQL **`IN`** 절 (목록 값 비교)**

SQL 쿼리에서 `IN`, `=`, `OR`는 **WHERE 절에서 데이터를 필터링하는 데 사용되는 키워드**이며, `=` 는 단일 값을 비교하고, `OR` 는  
여러 조건 중 하나라도 참이면 되며, `IN` 은 여러 값 목록 중 하나라도 일치하는 경우를 찾는 연산자로, `OR` 조건을 간결하게 만든다.

- **용도**: 특정 컬럼의 값이 주어진 값 목록중 하나라도 포함되면 해당 행을 반환  
  여러 개의 `=` 조건에 `OR`로 연결된 것과 같은 효과
- **예시**: 

  ```
  SELECT column1, column2, ...
  FROM table_name
  WHERE coulmn_name IN (value1, value2, ...);
  ```
- **특징**:
  - 여러 조건을 `OR`로 연결하는 것보다 간결하고 높은 가독성
  - `IN` 절 안에 다른 쿼리(서브쿼리) 결과를 사용하여 동적으로 값 목록 지정 가능
  - 값의 개수가 매우 많아지면(수백 개 이상), 오히려 성능이 저하될 가능성

참고: ***IN은 OR로 표현이 가능하나 OR은 IN으로 표현이 불가능하다.***

### IN vs OR 사용 예시 비교

```
-- OR 사용 예시 (긴 형태)
SELECT * FROM employees WHERE job_title = 'Manager' OR job_title = 'Analyst' OR job_title = 'Developer';

-- IN 사용 예시 (간결한 형태)
SELECT * FROM employees WHERE job_title IN ('Manager', 'Analyst', 'Developer');
```
