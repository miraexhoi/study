# AOP

Aspect-Oriented-Programming의 약자로 관점 지향 프로그래밍

***Aspect(관점)이란 흩어진 관심사들을 하나로 모듈화 한 것**을 의미
- OOP 모듈화의 핵심 단위가 "클래스"라고 한다면, AOP에서는 핵심 단위가 "관점"
- 반복되고 공통적으로 사용되는 부분을 **분리**함으로써 모듈성을 증가

OOP 클래스들은 보통 단일책임원칙에 따라 하나의 책임만을 갖게 설계된다. 

하지만 클래스를 설계하다보면 로깅, 트랜잭션 등 여러 클래스에서 공통적으로 사용하는 부가 기능들이 생긴다.  
→ 주요 비즈니스 로직은 아니지만, 반복적으로 여러 곳에서 쓰이는 데 이를 **흩어진 관심사 **라고 한다.

AOP 없이 흩어진 관심사를 처리하면 다음과 같은 문제가 발생한다.
- 여러 곳에서 반복적인 코드 작성
- 코드가 변경될 경우 여러 곳에 수정이 필요
- 주요 비즈니스 로직과 부가 기능이 한 곳에 섞여 가독성 저하

**따라서 흩어진 관심사를 별도의 클래스로 모듈화하여 위의 문제들을 해결하고,**  
**결과적으로 OOP를 더욱 잘 지킬 수 있도록 도움을 주는 것이 AOP이다.**
