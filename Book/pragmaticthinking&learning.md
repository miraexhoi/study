# `실용주의 사고와 학습` 을 읽고

### 02. 초보자에서 전문가에 이르는 여정

드라이퍼스 모델: 초보자 → 전문가로 가는 과정을 5가지로 구분 한 것

“전문성을 명확하게 말로 설명하기는 어렵지만 드라이퍼스는 기술단위로 적용 가능하다”

드라이퍼스 1단계: 초보자
- = 해당 기술 영역에서 사전 경험이 거의 없는 사람
- → 여기서 기술이란? 그 기술을 계속 쓰면서 사고에 변화가 생기는 것. 10년 차 개발자라도 1년치의 경험을 9번 반복했을 가능성
- 일을 해내기 위한 능력에 관심이 많음. 뭔가를 배우고 싶어하기보다 당면 과제를 달성하고자 함.
- “초보자는 레시피가 필요하다“
- 예) 옥수수 머핀 레시피에서 ‘대략 20분 요리’ 일 경우 더 오래/짧게 해야할때는?
- → “그건 ~이다, 라는 말의 의미가 무엇인가에 따라 다름” 이런 [ 무한회귀 ] 를 그만두어야.
- 규칙이 있으면 시작할 수는 있지만 더 나아갈 수는 없다.

드라이퍼스 2단계: 고급 입문자
- = 고정된 규칙에서 벗어나기 시작하지만 여전히 문제해결에 어려움 (+ 빠른 정보 욕구)
- 자신에 경험 중에서 비슷한 상황에 대한 조언을 활용하기 시작
- → 그러나 여전히 ‘큰 그림’은 잘 모른다
- + 더 큰 맥락을 보지 않고 자신과 상관 없는 일로 간주 (연관성 파악 어려움)

드라이퍼스 3단계: 중급자
- = 문제 영역에서 개념적인 모델을 정립하고 그 모델을 활용.
- 문제 스스로 해결 + 새로운 문제를 해결할 방법을 찾기 시작.
- 이전 단계에서 반사적으로 대응한 것과는 달리 문제를 ‘찾아서’ 해결.
- → 초보자에게는 멘토, 전문가의 시간 빼았지 않는 역할
- 자기 자신을 돌아보고 잘못을 스스로 교정 능력 x (애자일 방법론 적용 어려움)

드라이퍼스 4단계: 숙련자
- = 해당 기술에 관한 큰 개념적인 틀을 찾아서 이해 (단순한 정보 거부)
- “숙련자는 자가 교정을 할 수 있다”
- 이전에 잘못한 일을 스스로 교정 + 더 나은 접근법 → 더 이상 자기개발 한계
- ++ 직접 참여하지 않은 다른 사람의 경험에서도 배움
- → 격언(레시피 x 상황/맥락에 맞게 적용해야 하는 것)을 이해하고 적용 능력. 
- 참고 - 익스트림 프로그래밍 방법론 중 “뭐든지 잘못될 수 있는 것은 모두 테스트하라”

드라이퍼스 5단계: 전문가
- = 진화의 끝. 어떤 분야든 지식과 정보의 근원. 늘 더 나은 방법과 수단.
- “전문가는 이유가 아니라 직관으로 일한다”
- → 그러나 그들이 어떻게 결론을 내렸는지 그 과정을 
  완벽히 말로 설명하지 못함 → 그저 그것이 옳다고 느끼나 정말 이유를 모름
- 세부사항 중에서 중요한 것 / 중요하지 않은 것 가려내는 능력 → [ 범위 제한 + 패턴 발견 ]

전문가는 완벽하지 않다. (인지적 오류와 편견) → 같은 분야의 전문가들도 서로 다른 견해.

만약 어떤 영역에서 기술이 부족할 경우 오히려 스스로를 전문가라 생각.

“기술도 없고, 사실도 모른다: 자신의 부족함을 안다는 것이 어렵기 때문에 자신을 과대평가“

“무지가 지식보다 더 자주 확신을 준다” — 찰스 다윈

드라이퍼스의 오해: 고정된 규칙은 전문가의 생산성을 갉아 먹는다

→ TIP 1 — “초보자는 규칙을 활용하고 전문가는 직관을 활용하라”
- 규칙에 의지하던 것에서 직관에 의존하는 것으로 변화 / 인식의 변화
- 문제에서 유리된 관찰자가 아닌 전체 시스템의 일부로서의 사고 

+ TIP 2 — “네가 모른다는 것을 알라” (전문가 ≠ 선생님)

[전문성을 얻기까지 10년]

전문가가 되고자 한다면 어떤 분야든 10년가량의 노력을 쏟아부어야 한다.

그리고 그 과정의 노력도 중요. 단순 한 주제를 10년 간 반복 x (→ 연습의 필요성)

K.앤더슨 박사의 사려 깊은 연습 4가지 조건:
- 작업은 적당히 어려워야 함 → 도전적이지만 할 수 있는 수준
- 활동에 대해 풍부한 정보가 담긴 피드백을 받을 수 있는 환경
- 반복하면서 잘못을 교정할 기회 필요

[책임을 받아들이는 것]

태도의 변화를 이끌어내려면 기대치를 높여야 함.

그러나 고급 입문자는 이런 종류의 의사결정 어려움 (중급자의 기술 수준으로 올려야)

→ 이걸 달성하는 가장 좋은 방법: 같은 환경에 모범이 되는 사람

+ TIP 3 — “보고 따라 하면서 배우라“ (경험이 없으면 전문성도 없다)

[ 흉내 내라, 흡수하라, 혁신하라 ] — 트럼펫 주자 클라크 테리 (음악을 배우는 비법 중)

처음엔 다른 사람을 흉내 내다가 조금씩 암묵지와 경험 흡수 → 흉내를 넘어서 혁신할 수 있는 수준

‘수,파,리 훈련 주기’ 에 대하여?
- 수: 가르침 받은 대로 기술을 따라하며, 기술 변형 금지
- 파: 수련의 의미와 목적을 되새겨보고 더 깊이 있게 이해하려는 노력
- 리: 가르침을 넘어선 경험 초월 → 더 이상 학생이 아닌 새로운 방법을 창출해낼 수 있는 실천가

+ TIP 3 — “전문가를 남아 있게 만들도록 하라”

[도구의 함정을 경계하라]

형식화 모델을 지나치게 의존할 경우의 위험성 — 데보라 고돈 박사
- 모델과 현실을 혼동한다
- 정형화할 수 없는 것을 낮게 평가한다
- 개개인의 독립성을 훼손하는 행동을 법제화한다
- 경험 많은 실무자가 초보자 뒤차다꺼리를 한다
- 너무 많은 세부사항을 정의한다
- 복잡한 상황을 지나치게 단순화한다
- 지나친 일관성을 요구한다
- 맥락상의 뉘앙스에 대해 무감각해진다
- 규칙을 따르는 것과 현장에서 판단하는 것을 혼동한다
- 신비화

+ TIP 4 — “창의성, 직관, 고안력이 필요하다면 정형화된 방법론을 피하라“

[맥락에서 떨어진 객관성을 경계]

즉 무언가를 맥락에서 떼어놓고 객관적이려고 하는 것은 그 자체로 위험을 내재함

OOP 같은 시스템 사고에서는 종종 물체 자체보다 
물체 사이의 관계가 더 흥미 → 이런 관계가 모든 차이를 만들어내는 맥락을 형성

+ TIP 5 — “배우는 기술을 배우라”

드라이퍼스 모델의 교훈을 활용하여

더 많은 직관을 함양하고, 맥락과 상황적 패턴 관찰, 그리고 자신의 경험을 더 잘 활용하자.
